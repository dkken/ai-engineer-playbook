## 什么是AI系统设计面试

AI系统设计面试是2025-2026年新兴的面试类别，与软件系统设计和ML系统设计并列。面试官要求你设计一个AI系统（如RAG、Agent、评估系统），考察你的架构能力、技术选型和工程思维。

**出现频率**：60%的技术面试包含AI系统设计
**时长**：45-60分钟
**形式**：白板/在线协作工具 + 口头讲解

---

## 面试流程

### 1. 问题陈述（5分钟）

面试官给出一个场景：
- "设计一个企业级RAG系统"
- "设计一个多Agent协作平台"
- "设计一个AI输出质量评估系统"

### 2. 需求澄清（5分钟）

**必须问的问题**：
- "系统的目标用户是谁"
- "预期的数据量是多少"
- "预期的QPS是多少"
- "延迟要求是多少"
- "预算约束是什么"
- "是否需要实时更新"

**不要直接开始设计** —— 先澄清需求。

### 3. 高层架构（10分钟）

画出系统架构图：
- 主要组件
- 数据流
- 交互关系

### 4. 详细设计（20分钟）

深入关键组件：
- 数据流细节
- 技术选型
- 扩展性考虑
- 容错设计

### 5. 扩展和优化（15分钟）

讨论：
- "如果数据量增长10倍，怎么扩展"
- "如果用户量增长100倍，怎么调整"
- "成本怎么优化"
- "性能怎么优化"

### 6. 总结（5分钟）

回顾设计要点，回答面试官的总结问题。

## 常见设计题目

### 题目1：设计企业级RAG系统

**场景**：公司需要构建一个内部知识库问答系统，支持10万+文档，1000+用户。

**考察点**：
- 文档处理和分块策略
- 向量数据库选型
- 检索策略（语义搜索、混合搜索）
- LLM集成和Prompt设计
- 评估和监控
- 扩展性和成本

**追问示例**：
- "怎么处理PDF中的表格和图表"
- "怎么保证回答的准确性"
- "如果文档每天更新1000篇，怎么实时索引"
- "怎么控制LLM API成本"

### 题目2：设计多Agent协作系统

**场景**：公司需要构建一个客服Agent系统，能够处理复杂查询，必要时转人工。

**考察点**：
- Agent架构（规划、记忆、工具）
- 多Agent通信协议
- 状态管理
- 人在回路设计
- 安全护栏
- 监控和评估

**追问示例**：
- "怎么处理Agent死循环"
- "怎么保证Agent不泄露敏感信息"
- "如果工具调用失败，怎么降级"
- "怎么评估Agent的效果"

### 题目3：设计AI输出质量评估系统

**场景**：公司需要构建一个系统，自动评估AI输出的质量，包括准确性、安全性、相关性。

**考察点**：
- 评估指标设计
- 自动化评估流程
- 人工评估集成
- A/B测试框架
- 偏见检测
- 报告和监控

**追问示例**：
- "怎么定义'好的'AI输出"
- "怎么保证评估的客观性"
- "如果评估结果和人工判断不一致，怎么处理"
- "怎么检测偏见"

## 设计框架

### 4S框架

```
4S: Scenario, Service, Storage, Scale

1. Scenario（场景）
   - 明确需求
   - 定义用户
   - 确定约束

2. Service（服务）
   - 核心功能
   - API设计
   - 服务拆分

3. Storage（存储）
   - 数据模型
   - 数据库选型
   - 缓存策略

4. Scale（扩展）
   - 水平扩展
   - 负载均衡
   - 容错设计
```

### AI系统设计特殊考虑

```
AI系统 = 传统系统 + AI组件

传统系统考虑：
- 可用性
- 扩展性
- 一致性
- 延迟

AI组件考虑：
- 模型选择（API vs 自托管）
- Prompt管理
- 上下文窗口
- Token成本
- 幻觉处理
- 评估和监控
```

## 评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 需求理解 | 15% | 是否澄清了关键需求 |
| 架构设计 | 25% | 架构是否合理、清晰 |
| 技术选型 | 20% | 选型是否有理由、是否考虑替代方案 |
| 扩展性 | 15% | 是否考虑了扩展方案 |
| AI特定考虑 | 15% | 是否考虑了AI特有的问题 |
| 沟通表达 | 10% | 表达是否清晰、逻辑是否连贯 |

## 国内面试场景

### 国内大厂特点

- 更关注"为什么选这个技术"
- 可能会问"如果不用LangChain，怎么实现"
- 更关注成本优化（国内对成本更敏感）
- 可能会问"国产大模型怎么替代"

### 国内初创公司特点

- 更关注"快速落地"
- 可能会问"如果只有一周，怎么做MVP"
- 更关注"业务价值"
- 可能会问"怎么证明ROI"

## 准备策略

### 1. 掌握常见架构模式

- RAG架构（检索+生成）
- Agent架构（规划+记忆+工具）
- 评估架构（指标+自动化+人工）
- 微服务架构（服务拆分+通信）

### 2. 熟悉常用技术栈

- 向量数据库：Pinecone, Weaviate, Milvus, Chroma
- LLM框架：LangChain, LlamaIndex, Haystack
- Agent框架：LangGraph, CrewAI, AutoGen
- 评估工具：Ragas, DeepEval, Opik

### 3. 练习画架构图

- 在白板上练习
- 使用在线工具（Excalidraw, Draw.io）
- 确保能在5分钟内画出高层架构

### 4. 准备量化指标

- 延迟：P50, P95, P99
- 吞吐量：QPS, RPS
- 成本：每千次请求成本
- 质量：准确率, 召回率, F1

## 常见错误

### 错误1：直接开始设计，不问需求

❌ 直接画架构图
✅ 先问："用户是谁数据量多少延迟要求是多少"

### 错误2：过度设计

❌ 设计一个支持10亿用户的系统（实际需求是1万用户）
✅ 先设计MVP，再讨论扩展

### 错误3：忽略AI特有考虑

❌ 只设计传统系统，不提模型选择、Prompt、评估
✅ 明确说明AI组件的设计

### 错误4：没有备选方案

❌ "我选Pinecone因为它最好"
✅ "我选Pinecone因为托管方便，但Weaviate和Milvus也是备选，区别在于..."

---

## 关键结论

1. **AI系统设计是新兴面试类别** —— 需要专门准备
2. **先澄清需求，再设计** —— 不要直接画架构图
3. **考虑AI特有因素** —— 模型选择、成本、评估
4. **准备备选方案** —— 展示技术广度
5. **量化一切** —— 延迟、成本、质量都要有数字
