## 面试轮次分布

### 典型面试流程（海外）

| 轮次 | 出现频率 | 内容 |
|------|----------|------|
| 1. HR/Recruiter Screen | 95% | 基本情况、薪资期望、时间线 |
| 2. Hiring Manager Screen | 70% | 项目经验、技术背景、团队匹配 |
| 3. Technical Interview | 85% | 系统设计、项目深挖、编码 |
| 4. Take-Home Assignment | 45% | 实战项目（4-8小时） |
| 5. Final Interview | 60% | 行为面试、文化匹配、高管面 |

### 国内面试流程特点

| 轮次 | 出现频率 | 内容 |
|------|----------|------|
| 1. HR电话面 | 90% | 基本情况、薪资期望 |
| 2. 技术一面 | 80% | 项目经验、八股文、基础算法 |
| 3. 技术二面 | 70% | 系统设计、项目深挖 |
| 4. 技术三面/交叉面 | 40% | 跨部门技术评估 |
| 5. 总监/CTO面 | 50% | 架构能力、文化匹配 |
| 6. HR面（谈薪） | 80% | 薪资谈判、入职时间 |

**国内特有现象**：
- 算法题（LeetCode）出现频率仍较高（约60%），但呈下降趋势
- 八股文（基础知识问答）仍常见，但逐渐被项目深挖替代
- 部分大厂开始引入"实战项目"面试环节

## 各轮次详解

### 1. HR/Recruiter Screen（ recruiter筛选）

**时长**：15-30分钟

**考察点**：
- 基本情况匹配（经验、技能、地点）
- 薪资期望是否匹配预算
- 时间线（何时可以入职）
- 签证/工作许可（海外）

**准备建议**：
- 准备1-2分钟的自我介绍
- 明确薪资期望（给范围，不给具体数字）
- 了解公司背景和岗位需求
- 准备3-5个问HR的问题

**国内特殊注意**：
- 可能会问"为什么离职""职业规划"等常规问题
- 部分HR会询问"是否接受加班"
- 可能会问"是否有其他offer"

---

### 2. Hiring Manager Screen（ hiring manager筛选）

**时长**：30-45分钟

**考察点**：
- 项目经验深度
- 技术背景匹配度
- 团队文化匹配
- 解决实际问题的能力

**常见问题**：
- "介绍一下你最近的项目"
- "你在团队中扮演什么角色"
- "遇到过什么技术挑战，怎么解决的"
- "为什么对我们公司/岗位感兴趣"

**准备建议**：
- 准备2-3个项目的详细描述（STAR法则）
- 了解hiring manager的背景
- 准备问hiring manager的问题（团队、技术栈、项目方向）

---

### 3. Technical Interview（技术面试）

**时长**：45-60分钟

**考察内容**（按出现频率排序）：

| 考察类型 | 出现频率 | 说明 |
|----------|----------|------|
| 项目深挖（Project Deep Dive） | 75% | 对你做过的项目进行深入追问 |
| AI系统设计（AI System Design） | 60% | 设计RAG/Agent/评估系统 |
| 编码（Coding） | 50% | 算法题或AI相关编码 |
| 基础知识（Fundamentals） | 40% | LLM原理、RAG原理、向量检索 |
| 行为问题（Behavioral） | 35% | 团队协作、冲突解决 |

**重要趋势**：
- 项目深挖正在取代传统算法题成为主要考察方式
- AI系统设计成为独立面试类别
- 编码题从"纯算法"转向"AI工程化"（如：实现一个RAG检索函数）

---

### 4. Take-Home Assignment（实战作业）

**时长**：4-8小时（通常给3-7天时间）

**出现频率**：45%（海外）/ 20%（国内，但呈上升趋势）

**常见类型**：
- 构建一个RAG系统（最常见）
- 设计一个Agent工作流
- 实现一个评估框架
- 优化一个现有AI系统的性能

**评估标准**：
- 功能完整性（30%）
- 代码质量（25%）
- 工程化程度（20%）
- 文档和说明（15%）
- 创新性和扩展性（10%）

**准备建议**：
- 提前准备常用项目的starter code
- 注意代码规范和测试
- 写好README和运行说明
- 考虑边界情况和错误处理

**国内趋势**：
- 国内大厂（字节、阿里、腾讯）开始引入take-home assignment
- 部分公司用"现场编程"替代take-home
- 时间通常更短（2-4小时）

---

### 5. Final Interview（终面）

**时长**：30-60分钟

**考察点**：
- 文化匹配
- 长期职业规划
- 软技能（沟通、领导力）
- 对公司/产品的理解

**参与者**：
- 海外：VP/Director、HR、团队成员
- 国内：总监/CTO、HRBP

**准备建议**：
- 了解公司文化和价值观
- 准备"为什么选我们"的回答
- 准备问高管的问题（公司战略、技术方向）

## 面试轮次组合模式

### 模式1：标准模式（最常见）

```
HR Screen → Hiring Manager → Technical (项目+系统设计) → Take-Home → Final
```

出现频率：35%

### 模式2：精简模式

```
HR Screen → Hiring Manager + Technical (合并) → Final
```

出现频率：25%

### 模式3：深度技术模式

```
HR Screen → Technical 1 (项目深挖) → Technical 2 (系统设计) → Take-Home → Final
```

出现频率：20%

### 模式4：国内大厂模式

```
HR电话 → 技术一面（八股+算法） → 技术二面（项目+系统设计） → 总监面 → HR面
```

出现频率：15%

## 关键洞察

### 1. 项目深挖正在取代算法题

- 75%的技术面试包含项目深挖
- 只有50%包含传统编码题
- 趋势：算法题比例持续下降，项目深挖比例持续上升

### 2. AI系统设计成为独立类别

- 60%的技术面试包含AI系统设计
- 与软件系统设计和ML系统设计并列
- 考察点：RAG架构、Agent设计、评估策略

### 3. Take-Home Assignment越来越重要

- 45%的公司使用take-home
- 平均时长4-8小时
- 趋势：从"可选"变为"必须"

### 4. 国内面试仍偏重八股和算法

- 国内大厂仍有60%概率考算法题
- 八股文（基础知识）出现频率较高
- 但趋势也在向"项目+实战"转变

## 给求职者的建议

### 1. 准备项目深挖

- 准备2-3个项目的详细描述
- 每个项目准备5-10个可能被追问的问题
- 重点准备：技术选型、遇到的挑战、决策过程

### 2. 准备AI系统设计

- 熟悉RAG、Agent、评估系统的架构设计
- 练习在白板上画架构图
- 准备回答：扩展性、容错、监控、成本

### 3. 准备Take-Home

- 提前准备常用项目的starter code
- 注意代码规范和测试
- 写好README和运行说明

### 4. 国内求职特殊准备

- 算法题仍需要准备（LeetCode Medium）
- 八股文需要准备（LLM原理、RAG原理）
- 但重心应向"项目+实战"倾斜
