## 工程化改造任务

### 任务 1：部署与配置（1 小时）
- 使用 Docker 部署 giskard-oss 和 garak
- 配置本地 LLM（Ollama）替代默认 API
- 配置安全评估规则

### 任务 2：功能扩展（2 小时）
- 集成 garak 进行漏洞扫描
- 实现偏见检测
- 添加幻觉检测
- 实现安全护栏测试

### 任务 3：评估与优化（2 小时）
- 设计评估指标（安全分数、偏见指数、幻觉率）
- 对比不同安全策略的效果
- 优化安全评估效率

### 任务 4：生产化改造（1 小时）
- 添加错误处理和日志记录
- 配置环境变量管理（不硬编码 API Key）
- 编写部署文档和 README

## 常见坑点
- ❌ **坑点 1**：直接用默认配置部署，不配置本地模型，导致 API 费用过高
  - ✅ **解决**：配置 Ollama 本地 LLM（如 Qwen2.5）
- ❌ **坑点 2**：忽略偏见检测，只关注安全性
  - ✅ **解决**：集成 giskard-oss 偏见检测，覆盖性别、种族、年龄等维度
- ❌ **坑点 3**：不评估安全效果，盲目优化
  - ✅ **解决**：先建立安全评估 baseline（攻击成功率、偏见指数），再针对性优化
- ❌ **坑点 4**：安全评估通过了，但上线后还是出现有害输出
  - ✅ **解决**：建立线上监控机制，实时检测异常输出，触发自动降级
