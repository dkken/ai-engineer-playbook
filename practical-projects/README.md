## 项目结构

每个项目任务书包含以下章节：

1. **市场需求背景** — 为什么这个技能值得学（引用 job-market 数据）
2. **能力目标** — 做完后你能干什么
3. **基础开源项目** — 基于哪个开源项目做改造
4. **工程化改造任务** — 具体的任务步骤（从部署到生产化）
5. **评估标准** — 怎么判断自己做得好不好
6. **面试官考察维度** — 面试官会怎么追问这个项目
7. **常见坑点** — 别人踩过的坑和解决方案
8. **参考资源** — 官方文档、教程、社区

---

> **重要提示**：本项目提供的是任务规划和策略框架，不包含可运行代码。你需要根据任务书自行实现。开源项目迭代快速，部署前请查看官方文档的最新版本。

## 设计原则

1. **数据驱动**：每个项目的市场需求背景基于真实 JD 数据
2. **开源优先**：基于成熟开源项目，避免重复造轮子
3. **面试导向**：每个项目包含面试官考察维度和追问框架
4. **工程化**：强调从"能跑"到"能生产"的改造过程
5. **可评估**：每个项目有明确的评估标准和权重
