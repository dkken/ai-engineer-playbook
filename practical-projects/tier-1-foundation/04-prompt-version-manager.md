## 面试官考察维度

### 维度 1：Prompt 工程化思维（权重 35%）
- **考察点**：为什么需要 Prompt 版本管理？
- **追问示例**：
  - "Prompt 直接硬编码在代码中有什么问题？除了版本管理，还有什么风险？"
  - "如果 Prompt 中包含了敏感信息（如系统指令），怎么防止泄露给终端用户？"
  - "Prompt 版本回滚时，如何确保对应的模型版本、参数配置也一起回滚？"

### 维度 2：A/B 测试设计（权重 40%）
- **考察点**：如何设计有效的 A/B 测试？
- **追问示例**：
  - "A/B 测试的样本量怎么计算？如果流量很小（每天100次调用），怎么做统计显著性检验？"
  - "Prompt A 的准确率 85%，Prompt B 的准确率 87%，差异是否显著？怎么判断？"
  - "A/B 测试通过了，但上线后效果下降，可能是什么原因？（选择偏差、新奇效应）"

### 维度 3：成本优化（权重 25%）
- **考察点**：Prompt 优化如何降低成本？
- **追问示例**：
  - "Prompt 从 1000 tokens 优化到 500 tokens，成本降低了多少？对效果有什么影响？"
  - "什么情况下可以用更便宜的小模型替代 GPT-4？怎么验证小模型的效果？"
  - "缓存 Prompt 的响应结果，什么场景下有效？什么场景下无效？"

## 常见坑点
- ❌ **坑点 1**：Prompt 直接硬编码在代码中，无法管理
  - ✅ **解决**：使用 Git 管理 Prompt，实现版本控制；敏感指令放在服务端
- ❌ **坑点 2**：A/B 测试样本量不足，结果不可靠
  - ✅ **解决**：计算最小样本量（可用在线计算器），确保统计显著性（p < 0.05）
- ❌ **坑点 3**：忽略效果追踪，无法评估 Prompt 质量
  - ✅ **解决**：集成 opik 监控 Prompt 效果，关联版本号与效果指标
- ❌ **坑点 4**：Prompt 优化只关注效果，不关注成本
  - ✅ **解决**：建立"效果-成本"双维度评估，找到最佳平衡点
