## 一、市场需求技能重要性模型

### 设计逻辑：技能重要性 = 当前需求 × 增长预期 + 薪资溢价 + 层级保底

基于 [ai-engineering-field-guide](https://github.com/alexeygrigorev/ai-engineering-field-guide) 2,445 份 JD + [InterviewStack](https://www.interviewstack.io/) 3,449 份 JD + 国内公开报告综合分析：

#### 为什么不是单纯提及率？

单纯用 JD 提及率作为重要性指标存在明显缺陷：
- **Python 提及率 82.5%** → 但这是"基础必备"，不会则无法入行，而非"差异化优势"
- **Agent 提及率仅 14.4%** → 但增长率 +188%，2027 年多 Agent 协作将成为默认模式（AI Skill Market 预测）
- **AI 安全提及率 <5%** → 但国内合规需求增长 +125%，2028 年 80% 企业 AI 项目需嵌入隐私保护（今日头条 AI 趋势报告）

**类比股票估值**：就像 PE（市盈率）= 当前业绩 × 未来增长预期，技能重要性应该反映**市场预期**，而非仅当前需求。

#### 市场预期模型公式

```
综合重要性 = min(4, 基础分 × (1 + 增长率系数) + 溢价加分 + 层级保底)
```

| 参数 | 规则 |
|------|------|
| **基础分** | 提及率归一化到 0-4（82.5% → 3.3, 35.9% → 1.4） |
| **增长率系数** | >100%→0.8（市场极度看好）, 50-100%→0.5（看好）, 20-50%→0.3（温和看好）, <20%→0.1（稳定） |
| **溢价加分** | +$30K以上→+0.5, +$20K以上→+0.3 |
| **层级保底** | 基础层+0.5, 核心层+0.3, 进阶层+0.2, 专家层+0.1 |

#### 综合重要性评分表

| 技能领域 | 海外提及率 | 增长率 | 增长率系数 | 基础分×(1+系数) | 薪资溢价 | 层级保底 | 综合重要性(0-4) | 对应自测维度 |
|----------|-----------|--------|-----------|----------------|----------|----------|----------------|-------------|
| Python | 82.5% | +15% | 0.1 | 3.6 | 基准 | +0.5 | **3.5** | Python 编程 |
| RAG | 35.9% | +139% | 0.8 | 2.5 | +15% | +0.3 | **3.8** | RAG 系统 |
| LLM 应用开发 | 25.4% | +85% | 0.5 | 1.5 | +10% | +0.3 | **3.5** | LLM 应用开发 |
| 容器化与部署 | 62%综合 | +38% | 0.3 | 3.2 | +20% | +0.3 | **3.5** | 容器化与部署 |
| 软件工程 | 29.2% | +20% | 0.1 | 1.3 | 基准 | +0.5 | **3.0** | 软件工程基础 |
| Prompt Engineering | 29.1% | +45% | 0.3 | 1.6 | +5% | +0.3 | **3.2** | Prompt Engineering |
| Agent 开发 | 14.4% | +188% | 0.8 | 1.1 | +25% | +0.2 | **3.5** | Agent 开发 |
| MLOps/可观测性 | 18.5% | +95% | 0.5 | 1.1 | +30% | +0.2 | **3.5** | MLOps 与可观测性 |
| 模型评估与优化 | 11.1%显性 | +178% | 0.8 | 0.8 | +20% | +0.2 | **3.2** | 模型评估与优化 |
| 数据工程 | 22.7% | +32% | 0.3 | 1.2 | +$24K | 0 | **2.8** | 数据工程 |
| 系统架构 | ~18% | +28% | 0.3 | 0.9 | +$34K | +0.1 | **3.0** | 系统架构 |
| AI 安全与对齐 | <5% | +125% | 0.8 | 0.4 | +25% | +0.1 | **3.0** | AI 安全与对齐 |

**结果解读**：
- **3.8 分（RAG）**：当前热门 + 高增长预期，市场最看好的技能
- **3.5 分（Python/LLM/部署/Agent/MLOps）**：五足鼎立，各有优势——Python是根基，LLM/Agent是未来，MLOps是生产刚需
- **3.2 分（Prompt/评估）**：核心层基础，稳步增长
- **3.0 分（软件工程/架构/安全）**：软件工程是保底，架构和安全是高级岗位溢价来源
- **2.8 分（数据工程）**：支撑层，高级岗位需要

**关键洞察**：
- 增长率 >100% 的技能（Agent、RAG、评估、安全）全部达到 3.0+，体现市场对未来高预期
- 薪资溢价高的技能（MLOps +30%, 架构 +$34K）也达到 3.0+，体现稀缺性价值
- 没有技能低于 2.8，说明 AI 工程师需要**全面发展**，不存在"可忽略"的技能

## 二、差距计算方法

### 公式

```
差距指数 = (市场需求权重 × 满分) - (市场需求权重 × 实际得分)
优先级 = 差距指数 × 市场增长率
```

### 市场增长率（2024-2026）

| 技能领域 | 增长率 | 趋势 | 数据来源 |

| RAG | +139% | 稳步增长 | job-market/02-skill-demand-analysis.md |

| MLOps/可观测性 | +95% | 快速增长 | 生产需求驱动 |
| LLM 应用开发 | +85%（国内） | 快速增长 | 国内大模型应用爆发 |
| Prompt Engineering | +45% | 稳步增长 | job-market/02-skill-demand-analysis.md |

## 五、常见差距模式

### 模式 1：技术强，工程弱

**特征**：Python/RAG/LLM 得分高，Docker/CI/CD/MLOps 得分低
**风险**：面试时能讲清楚原理，但无法展示可运行的项目，缺乏生产经验
**建议**：优先完成 P2-03（AI 代码审查机器人）+ P3-03 监控模块，强化工程化和可观测性能力

### 模式 2：工程强，AI 弱

**特征**：Docker/K8s/软件工程得分高，RAG/Agents/LLM 应用开发得分低
**风险**：有工程能力但不懂 AI 场景，无法独立设计 AI 系统
**建议**：优先完成 P1-01（RAG 知识库）+ P1-02（Agent 任务调度器），建立 AI 基础

### 模式 3：全栈均衡，无亮点

**特征**：各维度得分中等（2-3 分），无突出短板也无突出优势
**风险**：简历缺乏差异化，在竞争中容易被淹没
**建议**：选择 1-2 个高增长率技能深入（AI 安全 P2-04 或 MLOps P3-03 监控），建立差异化优势

### 模式 4：理论强，实践弱

**特征**：概念题得分高（LLM 原理、RAG 流程），动手题得分低（部署、优化、故障处理）
**风险**：面试时被追问细节就露馅，无法证明生产级能力
**建议**：每个项目至少完成"部署 + 一个功能扩展 + 监控告警"，积累可展示的成果

### 模式 5：国内求职，缺国产化技能

**特征**：海外技术栈熟悉（OpenAI、AWS），国产大模型和国内合规不了解
**风险**：国内岗位竞争力下降，尤其国企和合规敏感行业
**建议**：补充国产大模型 API 调用（通义/文心/智谱至少 1 个），了解算法备案和数据安全法

## 六、动态更新

本分析会根据以下数据定期更新：

1. **JD 数据更新**：每季度更新技能需求权重
2. **用户反馈**：收集自测数据，校准差距计算
3. **市场趋势**：跟踪新兴技能（如 MCP、知识图谱、多模态 RAG）
4. **国内政策**：跟踪国产大模型发展和合规要求变化
