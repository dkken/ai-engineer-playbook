# 面经：Meta Software Engineer - Machine Learning (E4)

> **来源**：Reddit，late 2025
> **岗位**：Meta Software Engineer - Machine Learning (E4)
> **面试者背景**：有 ML 工程经验，熟悉推荐系统
> **结果**：通过

---

## 面试流程

| 轮次 | 形式 | 时长 | 内容 |
|------|------|------|------|
| 1 | 电话面试 | 45 min | 数组问题 + 基础 ML |
| 2 | Onsite | 全天 | 4-5 轮技术面试 |

**总时长**：约 2-3 周

---

## 题目记录

### 电话面试（Round 1）

**题目 1**：Kth Largest Element in an Array
- 类型：经典 Top-K 问题
- 考察点：堆、快速选择

**题目 2**：Max Consecutive Ones
- 类型：数组遍历
- 考察点：滑动窗口、边界处理

**面试者反馈**：
- "Phone screen with array problems."
- 难度中等，注重代码正确性

### Onsite（Round 2-5）

**DSA 轮**：
- 图遍历变体（与社交图谱相关）
- 考察点：图的表示、BFS/DFS、优化

**AI-assisted Coding 轮**：
- "Finish tasks in unfamiliar codebase."
- 考察点：在陌生代码库中快速理解和修改代码
- 可能使用 AI 工具辅助

**ML System Design 轮**：
- 推荐系统设计
- 考察点：特征工程、模型选择、在线服务

**行为面试轮**：
- STAR 故事
- 考察点：协作、失败经历、项目 ownership

---

## 面试者复盘

### 做得好的

1. **代码正确性**：优先保证代码通过测试
2. **沟通**：主动解释思路，与面试官互动
3. **ML 经验**：有实际的推荐系统经验

### 可以改进的

1. **代码速度**：可以更快地完成实现
2. ** unfamiliar codebase**：在陌生代码库中导航的速度

### 建议

- "prioritize passing tests over perfect code; explain trade-offs verbally."
- 练习在陌生代码库中快速定位问题
- 准备推荐系统的深度案例
