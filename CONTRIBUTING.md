# 贡献指南

感谢你对 AI Engineer Playbook 的兴趣！这本 Playbook 需要社区的力量才能持续进化。

## 贡献方式

### 提交面经

分享你的真实 AI 工程师面试经历（可匿名）。格式参考 [resources/interview-experiences/README.md](resources/interview-experiences/README.md)。

### 补充实战项目

设计新的项目任务书。提交到 `practical-projects/` 对应层级目录，单个 Markdown 文件。

**项目任务书模板**：

```
practical-projects/tier-X/
└── XX-your-project-name.md
```

必须包含：
- [ ] 市场需求背景（引用 job-market 数据）
- [ ] 能力目标和适合考察的能力
- [ ] 基础开源项目信息（名称、Stars、技术栈、许可证）
- [ ] 至少 4 个工程化改造任务
- [ ] 评估标准（至少 4 个维度 + 权重）
- [ ] 面试官考察维度（至少 3 个 + 追问示例）
- [ ] 常见坑点（至少 2 个 + 解决方案）
- [ ] 参考资源链接

### 贡献市场数据

- 通过 Issue 提交匿名 JD 样本
- 标注数据来源（公司类型、地区、时间）
- 不编造数据

### 贡献资源

- tools-comparison/：提供技术选型对比分析
- case-studies/：提交生产案例（需有公开来源）
- awesome-ai-engineering-cn.md：提交高质量中文资源链接

## 翻译规范

- 翻译自原版项目的内容需标注原文出处
- 技术术语保留英文并附中文解释
- 国内数据和观察需标注具体来源

## 行为准则

- 尊重他人，保持专业
- 不接受抄袭内容
- 数据来源必须诚实标注
- 不提交包含个人敏感信息的真实公司数据（面经可以匿名）

## 提交流程

1. Fork 本仓库
2. 创建你的功能分支
3. 按上述模板编写内容
4. 提交 PR，在描述中说明：
   - 你贡献了什么
   - 信息来源是什么
   - 是否经过验证（如是面经：是否来自真实经历；如是项目：是否部署验证过）

---

> **提示**：如果你在实际部署项目时遇到了任务书中没提到的问题，欢迎提交 Issue 分享经验，帮助社区完善内容。
